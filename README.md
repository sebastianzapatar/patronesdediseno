Bienvenido al repositorio educativo de **Patrones de Diseño** y **Principios SOLID**, desarrollado para la asignatura de Ingeniería de Software de la Universidad EIA.

## 🌟 Contenido del Repositorio

Este proyecto está dividido en dos grandes pilares de la ingeniería de software:

### 1. Principios SOLID (`src/patrones/principios/solid/`)
Las reglas fundamentales para construir software escalable, modular y mantenible. Cada principio cuenta con un ejemplo práctico documentado línea por línea:
- **S** - Responsabilidad Única (SRP)
- **O** - Abierto/Cerrado (OCP)
- **L** - Sustitución de Liskov (LSP)
- **I** - Segregación de Interfaces (ISP)
- **D** - Inversión de Dependencias (DIP)

### 2. Patrones de Diseño (`src/patrones/`)
Soluciones arquitectónicas probadas a problemas comunes, divididas en sus categorías formales:
- **🛠️ Creacionales:** `Singleton`, `Factory Method`, `Builder`
- **🏗️ Estructurales:** `Adapter`, `Decorator`
- **🧠 Comportamiento:** `Observer`, `Strategy`
- **⚡ Resiliencia (Infraestructura):** `Circuit Breaker`

## 🚀 Cómo ejecutar el código

1. Todo el código está escrito en **Java** puro.
2. Cada patrón tiene su propio paquete aislado.
3. Para probar cualquier patrón, simplemente busca su clase principal (por ejemplo, `MainSingleton.java`, `MainStrategy.java`, etc.) y ejecútala. 
4. El código no contiene concurrencia avanzada (`Threads`) con el fin de mantener los conceptos simples, limpios y completamente enfocados en la academia.
